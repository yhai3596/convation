# 技术决策记录

## D1 · 2026-07-15 · Express+EJS 服务端渲染，不用 Next.js/SPA
设计交付是纯 HTML 高保真稿，EJS 可近乎原样移植（保真度最高、无构建步骤、部署最简）。
交互点（问卷/助手/后台页签）用原生 JS 足够。README 建议的 Next.js 记为可选演进方向。

## D2 · 2026-07-15 · SQLite 替代 PostgreSQL
单机个人平台，better-sqlite3(WAL) 零运维、单文件备份；数据模型完全按交付 README 的 backend spec。
若未来并发/多机需要，schema 可平移 Postgres。

## D3 · 2026-07-15 · 字体自托管，不引 Google Fonts
主要访客在国内，fonts.googleapis.com 不可达会阻塞渲染。fontsource 包按 unicode-range 分片，
postinstall 仅复制 CSS 实际引用的 400/600 分片（6.1MB，剔除近 100MB 的 CJK 整包）。

## D4 · 2026-07-15 · Agent 能力"模板优先，LLM 可选"
诊断报告/助手/评论自动回复默认走确定性规则+FAQ（与设计稿演示口径一致、离线可用），
配置 Z_AI_API_KEY 后自动升级 LLM 生成（站内知识约束、超时回退）。原因：不把用户 API Key
写进代码/仓库；无 Key 时功能依然完整诚实（后台明示"模板模式/已连接 LLM"）。

## D5 · 2026-07-15 · 部署走"新加坡服务器 + GitHub autopull"（复用 hvac 模式）
国内服务器无 ICP 备案会被拦 80/443；该新加坡机已验证可服务国内访客（hvac.geopro.cc）。
本机 SSH 到该服务器被网络路径拦截（hvac 运维记录已定性），故首次引导由用户在腾讯云
控制台网页终端跑一条脚本，之后 push 即部署。DNS 记录创建放进服务器端脚本
（用服务器上 acme.sh 已存的 GoDaddy 凭据），凭据不经过本机会话。

## D6 · 2026-07-15 · 生产种子只含"站点内容"，不含演示数据
文章/课程/工具/案例按设计稿文案种子化（属站点内容，后台可改）；
演示用户/评论/访问事件仅 SEED_DEMO=1（本地）注入，避免线上出现虚构社交证明。
诊断页"报告已发送至邮箱"文案按 SMTP 是否配置自动切换措辞，保持诚实。
